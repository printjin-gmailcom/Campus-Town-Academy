CRM 마케팅 (Customer Relationship Management)

: 친구가 되는 것

: 적시에 적합한 메세지 전달

: 퍼포먼스 마케팅의 역할이 줄어들면서 필요성 증가

: CAC(고객 획득 비용) 증가, ROAS(광고비에 대한 매출 비율), LTV(생에 가치) 감소 > 리텐션 개선

: 서드파티(고객으로부터 직접 수집한 데이터가 아닌 제3의 주체로부터 간접적으로 얻는 데이터) 데이터 활용의 제약 > 퍼스트 파티 고객 데이터 활용

: 시장의 포화, 똑똑해진 고객 > 넛지(부드러운 권유, 유연하고 비강제적 접근, 선택의 자유 침해 X, 경제적 인센티브 변화 X)로 자연스러운 행동 유도



CRM 마케터

: 선택적 설계자

: 통찰과 분석, 고객에 대한 관심 필요.

: 간단한 역발상과 작지만 파워풀한 아이디어 



CRM의 목적

: LTV(고객 생애 가치)를 개선시키는 것

: 푸쉬메세지 하나하나 오픈율이나 전환율보다 요소 BY 요소 개선에 집중



CRM 업무 프로세스

1. 거시적 목표 정의 : KPI 정의 : 궁극적으로 서비스가 달성하고자 하는 가장 큰 목표

2. 미시적 목표 정의 : 북극성 지표 - 거시적 목표 달성을 위해 개선해야하는 인과지표 : 거시적 목표룰 달성하기 위한 세부 목표, CRM 캠페인에서 전환 목표가 됨

3. CRM 시나리오 작성 : 앱 내 고객 여정을 시나리오로 설계

4. 이벤트 설계 : 이벤트 기획 및 설계 & 데이터 정합성 확인

5. 캠페인 셋팅



목표

: 거시적 목표 - 궁극적으로 서비스가 달성하고자 하는 가장 큰 목표 = KPI

: 미시적 목표 - 세부 목표, 거시적 목표와 인과관계 = 북극성지표



KPI (Ker Performance Index)

: 핵심성과지표

: 구체적 세부적 독립적 작성



매출 != KPI

매출 = 구매자수 * 객단가 * 재구매율



구조화 

: 전체를 잘 나누어서 집중할 범위를 정하는 것

: 넓은 과제를 좁히는데 효과적 = 집중할 그 문제를 발견할 수 있음

: 의사결정을 빠르고 정확하게 도움



개념화

: 특정 용어를 사용할 때 무엇을 의미하는지 구체적으로 정의하는 것

: 데이터 개념화 할 때는 시간과 단위 고려



집중할 범위 정하기



서비스 성장 단계

: 이전 성공 요인이 일정 시점 이후 유효하지 않게 되는 것 = 골짜기

> 완전히 새로운 차원의 문제 해결이 필요

: 단계마다 KPI의 변화가 필요



캐즘(Chasm)

: 새로운 기술이 대중적으로 보급되기까지 걸리는 시간



북극성 지표 = 미시적 목표 

== 아하 모먼트 = 제품이 고객에게 전달하는 핵심 가치가 반영된 지표

: 깊이, 넓이, 빈도, 효율 등

: 트위터 팔로워 30명, 페이스북 친구 10명, 슬랙 메세지 2000개 등등



매출은 서비스의 핵심 가치를 반영하지 못함

== 결과일뿐, 무엇을 해야하는지 알 수 없음

CRM 시나리오

: 유저의 주요 인앱 액션에 따른 CRM 발송 여부를 도식화



플로우 차트

: 어떠한 일을 처리하는 과정을 순서대로 간단한 기호와 도형으로 도식화한 것

: 프로세스 처리 순서를 포함한 단계 간의 상호관계 표시

: 단순화하며 쉽게 이해 가능



MECE (Mutually Exclusive Collectively Exhaustive)

: 겹치지 않으면서 빠짐없이 나눈 것

: 한가지 기준으로 나눔 > 한가지 기준으로 쪼갬

: 누락 중복 자제

: 불필요한 자원 소모는 막고 효율은 극대화



CRM 시나리오는 시간 개념 없음 ㅠ



유량 = 일정 기간, 저수지에서 흘러 나온 물 & 흘러 들어온 물

저량 = 일정 시점, 저수지에 고여 있는 물 



유저의 행동의 변화를 시간에 흐름에 따라 주기적으로 체크

클라이언트

: 서버에게 서비스 요구하는 사용자나 컴퓨터

: 사용자의 입력을 처리

 

서버

: 서버프로그램이 실행되느 있는 하드웨어

: 클라이언트의 요청을 받아서 처리하고 이를 다시 클라이언트에 응답

 

 

SDK

: 소프트 웨어 개발 키트

: 앱을 동작시키기 위한 각종 코드와 문서, API가 정리되어있는 파일

 

 

API

: Application Programming Interface

: 클라이언트와 서버가 서로 상호작용하는 것을 도와주는 매개체

: 레스토랑의 점원 같은 역할

 

 

 

전송주기

: 실시간 OR 배치

 

데이터 종류

: 고객 행위 정보

: 이벤트 설명하는 부가 정보

: 고객 개인정보

: 영어로 작성, 직관적 이해 가능하도록 작성

 

 

메타데이터
